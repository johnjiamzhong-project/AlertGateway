## 长期模型优化方向（需新架构/新数据集才可重启）

由于前期对六类数据集（train/val/test=372/30/29）尝试了直接通道裁剪（3.6% 与 14.6% 剪枝）、C2f 深度缩减（80x80 stage 裁剪）、ReLU6 模块替换以及知识蒸馏（mAP50=56.82%），全部因精度显著退化且未达到验收门槛（test mAP50 >= 74.2%）而被淘汰。因此，图优化和轻量化路线处于阶段性饱和状态。

后续若要重启模型图级优化，需满足以下条件之一：
- **新架构设计**：探索如带有更小/更深特征提取能力的轻量 Backbone（如 MobileNetV4 / RepVGG）或自定义小参数量 YOLO 检测头，设计能更好在 RKNPU 上做算子融合的图。
- **更大规模的数据集**：当前桌面六类数据集规模较小（数百张图片），不足以支撑高比例的直接剪枝微调或轻量学生模型的强泛化蒸馏。需要采集并人工标注至少 5000+ 张多场景桌面物品图片，增加数据多样性后，才能重启剪枝与蒸馏。
- **新微调策略**：探索更长 Epoch 的微调（如 100+ Epoch）、更细致的学习率衰减计划和多尺度随机增强，以对抗剪枝带来的表征能力衰退。

*说明：后处理循环优化（L1/L2 缓存优化）已于 2026-07-11 正式部署至板端生产二进制，后处理延迟降低 1.297 ms（中位数 4.139 ms -> 2.842 ms），相关待办已全部完成并清理。通过在 benchmark 中引入阶段级计时基线，测得除类别筛选外的其余阶段（DFL解码 0.135 ms，坐标映射 0.009 ms，NMS 0.017 ms）耗时总和仅 0.161 ms，远低于 0.3 ms 优化门槛。因此已正式明确关闭 CPU 后处理的进一步优化。*
