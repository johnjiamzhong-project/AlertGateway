## 一、H.264 硬解验证

生成测试文件（yuv420p，MPP 不支持 yuv444p）：

```bash
ffmpeg -f lavfi -i testsrc=duration=5:size=1280x720:rate=30 \
       -c:v libx264 -pix_fmt yuv420p /tmp/test_h264_420.mp4
```

硬解：

```bash
/opt/ffmpeg-rockchip/bin/ffmpeg \
  -c:v h264_rkmpp \
  -i /tmp/test_h264_420.mp4 \
  -f null -
```

结果：`nv12` 输出，`speed=7.89x`，VPU 正常工作。

## 三、RTMP 拉流硬解播放

Windows 端用 SRS（192.168.1.5）作推流服务器，推测试流：

```powershell
ffmpeg -re -f lavfi -i testsrc=size=1280x720:rate=30 `
  -c:v libx264 -pix_fmt yuv420p -preset ultrafast `
  -f flv rtmp://192.168.1.5/live/test
```

板子拉流验证解码（无显示）：

```bash
LD_LIBRARY_PATH=/opt/ffmpeg-rockchip/lib \
/opt/ffmpeg-rockchip/bin/ffmpeg \
  -vcodec h264_rkmpp \
  -i rtmp://192.168.1.5/live/test \
  -f null -
```

结果：`h264_rkmpp` 解码，`fps=50`，`nv12` 输出。

板子拉流播放（需重新编译 ffmpeg-rockchip 加 SDL2）：

```bash
# 板子上安装 SDL2
sudo apt install -y libsdl2-dev

# 重新编译 ffplay
cd ~/build/ffmpeg-rockchip-master
./configure \
  --prefix=/opt/ffmpeg-rockchip \
  --enable-gpl --enable-version3 \
  --enable-libdrm --enable-rkmpp \
  --enable-libx264 --enable-libx265 \
  --enable-libvpx --enable-libopus \
  --disable-static --enable-shared \
  --enable-ffplay --enable-sdl2
make -j$(nproc) && sudo make install
```

播放：

```bash
LD_LIBRARY_PATH=/opt/ffmpeg-rockchip/lib \
/opt/ffmpeg-rockchip/bin/ffplay \
  -vcodec h264_rkmpp \
  -i rtmp://192.168.1.5/live/test
```

结果：显示器正常显示画面。

## 四、HTTP-FLV 拉流硬解播放

SRS 默认开启 HTTP-FLV（端口 8080）：

```bash
LD_LIBRARY_PATH=/opt/ffmpeg-rockchip/lib \
/opt/ffmpeg-rockchip/bin/ffplay \
  -vcodec h264_rkmpp \
  -i http://192.168.1.5:8080/live/test.flv
```

结果：显示器正常显示画面。

## 注意事项

- MPP 只支持 `yuv420p`，`yuv444p` 会报 `Function not implemented`
- 系统自带 ffmpeg 4.4.2 没有 rkmpp，必须用 `/opt/ffmpeg-rockchip/bin/ffmpeg`
- 运行时需指定 `LD_LIBRARY_PATH=/opt/ffmpeg-rockchip/lib`，或写入 `/etc/ld.so.conf.d/`
- ffplay 默认编译不含 SDL2，需额外安装 `libsdl2-dev` 后重新编译
