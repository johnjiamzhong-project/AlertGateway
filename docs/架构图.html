CaptureThread 内部流程

  1. open_device()：open fd → VIDIOC_S_FMT 设分辨率/YUYV → VIDIOC_S_PARM 设帧率
  2. init_buffers()：VIDIOC_REQBUFS 申请 4 块内核缓冲区，mmap 映射到用户空间
  3. start_stream()：全部缓冲区 VIDIOC_QBUF 入队 → VIDIOC_STREAMON
  4. run() 主循环：select 等待可读 → DQBUF 取一帧 → assign 拷贝到 Frame::raw_data → 投递 enc_queue（阻塞）和 infer_queue（非阻塞）→ 立即 QBUF 归还
  5. 停止时：stop_stream() STREAMOFF → close_device() munmap + close fd

InferThread 内部流程

  1. load_model()：加载 rknn 模型，识别双输出张量索引（box_output_idx_ / cls_output_idx_）
  2. 预分配 input_mem_（NPU DMA 零拷贝输入缓冲区）
  3. 每轮 run()：从 infer_queue 取最新帧（排空旧帧）→ RGA 硬件预处理（颜色转换+缩放，直写 DMA 缓冲，耗时 ~1ms）
  4. rknn_run() 推理（~30ms，已对标官方 demo 验证为硬件极限）
  5. YoloPostprocess：box 反量化（0~640）+ cls 反量化（0~1）→ NMS → target_classes 过滤
  6. 结果写入 shared_dets_.set()；若与 last_summary_ 不同且超过 report_interval_sec，调 summarize() 生成 JSON 并 push 到 mqtt_queue

EncodeThread 内部流程

  1. init_mpp()：创建 MppCtx / MppApi，配置 H.264 编码参数（GOP=8，Main profile，level 3.1，bitrate）
  2. 每轮 run()：从 enc_queue 取一帧 YUYV
  3. yuyv_to_nv12()：Y 平面逐字节取，UV 平面从偶数行取样完成 2× 垂直降采样（无 RGB 中间缓冲）
  4. shared_dets_.get() 拷贝最新检测结果 → draw_rect_nv12()（内部调 draw_hline_nv12 / draw_vline_nv12）直接在 NV12 平面画绿色框（BT.601 Y=145,U=54,V=34）
  5. 送入 MPP 编码器，取出 EncodedPacket（含 pts/dts/is_keyframe）push 到 stream_queue

StreamThread 内部流程

  1. open_rtmp()：创建 AVFormatContext，连接 RTMP URL
  2. 每轮 run()：从 stream_queue 取一个 EncodedPacket
  3. parse_annexb() 拆出 NALU 列表；首个 IDR 帧时提取 SPS/PPS 调 write_extradata()
  4. write_packet() 按 pts/dts 写入 FLV tag 推流
  5. 连接失败或写入异常时进入 reconnect_loop()，定时重试 open_rtmp() 直到恢复，期间不阻塞队列消费（丢弃或等待视实现而定）
队列 / 共享状态一览
名称类型容量生产者 → 消费者push 策略
enc_queueBlockingQueue<Frame>1（~68ms 缓冲）Capture → Encode阻塞 timeout=100ms，每帧必达
infer_queueBlockingQueue<Frame>2Capture → Infer非阻塞 timeout=0，忙时丢帧
stream_queueBlockingQueue<EncodedPacket>2（~137ms 缓冲）Encode → Stream默认阻塞
mqtt_queueBlockingQueue<std::string>16Infer → Mqtt默认阻塞，容量大避免反压推理
SharedDetectionsmutex + vector—Infer 写 → Encode 读set() / get()，无队列阻塞语义
关键设计点
  1. 采集与推理彻底解耦：CaptureThread 同时投递两条队列；enc_queue 阻塞保证编码帧不丢、不被推理速度拖累；infer_queue 非阻塞允许丢帧，InferThread 永远处理"最新"而非"排队中最旧"的帧。
  2. 检测框跨线程传递走共享区而非队列：SharedDetections 用 get() 返回值拷贝而非引用，避免 EncodeThread 持锁时间过长，代价是编码端可能叠加"上一次"推理结果（15fps 下视觉无感）。
  3. 零拷贝两端：采集端 V4L2 mmap 直读内核缓冲区；推理端 RGA 预处理直接写 NPU DMA 输入（input_mem_），均跳过 CPU 数据搬运。
  4. 启动/关闭顺序均为"下游先、上游后"：启动时 mqtt → stream → encode → infer → capture；关闭时反向，先停 capture 再关队列让下游感知 EOF 自然退出，避免半启动 / 写已关闭流的竞态。
  5. DetectionReporter 是死代码：src/detect/ 目录文件仍在仓库但未被 CMakeLists.txt 引用，功能已被 InferThread 吸收（画框移到 EncodeThread，汇总上报移到 InferThread::summarize()）。
