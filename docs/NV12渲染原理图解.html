为什么用 YUV 不用 RGB

Y = 亮度，U/V = 色度。人眼对亮度变化敏感、对颜色变化不敏感，所以可以让亮度不打折，颜色信息打折存，肉眼几乎看不出差别，但能省下大量数据量。

4:2:0 子采样

每个像素都有自己的 Y 值，但每 2×2=4 个像素共享一组 U、V 值。颜色信息被压缩到 1/4，这就是下面演示要可视化的核心规则。

NV12 = YUV420SP

把 Y 和 UV 分成两个连续的"平面"前后存：Y平面(w×h字节) + UV平面(w×h/2字节，U/V交替存)，总大小 w*h*3/2，对应 EncodeThread.cpp 里 frame_size 那行。
交互演示：像素坐标 → UV平面字节偏移量

把鼠标移到下面任意一个像素格上 —— 同色的 2×2 像素块共享同一组 UV，移到 UV 平面里对应的两个字节（U 蓝、V 红）会一起高亮，下方面板实时算出对应的内存偏移量。

src/encode/EncodeThread.cpp · draw_hline_nv12（节选）

// y_plane[y*w+x] 就是上面像素网格里每格标的"Y偏移"
y_plane[y * w + x] = yv;

// ux：把 x 拉回到它所在2像素块的起始列——上面UV网格里每对格子的左边那一列
int ux = x & ~1;

// (y>>1)：UV平面的行号，每2行像素共享1行UV——上面UV网格的"行"
uv_plane[(y >> 1) * w + ux]     = uv;  // U，蓝色那格
uv_plane[(y >> 1) * w + ux + 1] = vv;  // V，红色那格，紧挨着U存
