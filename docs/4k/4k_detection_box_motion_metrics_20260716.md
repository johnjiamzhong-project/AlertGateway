# 4K 检测框抖动与滞后量化记录（2026-07-16）

## 目标与口径

本轮先建立可重复统计，不直接扩大 TTL、死区或使用速度外推。诊断只处理连续两次都存在的
相同 Active Track，分别计算这些框的原始/显示稳健中心与中位宽高，避免目标数量变化时直接
比较全部框中心。

`tools/analysis/analyze_motion_stats.py` 将样本按原始整体中心速度分类：

- 低速：不高于 0.35 个框对角线/秒；
- 运动：不低于 0.75 个框对角线/秒；
- 抖动：低速区间单步中心/尺寸变化的 RMS、P90；
- 等效滞后：原始中心与显示中心沿运动方向的距离除以当前整体中心速度；
- 连续性：方向反转、轨迹集合变化、新建/丢失轨迹率；
- 链路：编码 FPS、码率、PTS 差及丢帧/失败计数。

等效滞后用于同源 A/B，不等于摄像头到播放器的端到端延迟；原始 YOLO 框也不是真值，候选
最终仍需人工画面验收。

## 首轮基线

一次 67 秒诊断得到 1320 个有效样本：低速中心抖动 RMS 为框对角线的 0.74%，当前显示滤波
相对原始框降低 19.29%；低速尺寸抖动 RMS 为 1.27%，降低 43.60%；运动等效滞后 P90 为
39.55ms。尺寸变化直接作用于四条边，与“中心相对稳定、边框抖动明显”的观察一致。

## size alpha 0.45 / 0.30 A/B

两组均从同一 4K 文件起点发布，分辨率、30FPS、18Mbps、中心滤波、关联、TTL、三帧 PTS
对齐和固定输出地址完全相同；候选只把 `track_size_alpha_max` 从 0.45 调为 0.30。完整报告见
`runs/testsrc2/motion_ab_size30_20260716.md` 和对应 JSON。

- 低速尺寸抖动 RMS：1.28% → 1.01%，改善 21.34%；
- 全段显示框单步跳变 P95：5.55% → 4.74%，改善约 14.6%；
- 运动单步跳变 P90：7.57% → 8.12%，退化 7.24%；
- 运动等效滞后 P90：33.67ms → 39.74ms，退化 18.03%；
- 输出 FPS P50：30.03 → 30.00，编码/推流失败仍为 0。

该 A/B 仍不够严格：30FPS 最新帧覆盖会让两次运行选择不同推理帧，两组运动样本数相差
17.2%，编码输入丢帧累计分别为 12/27。中心参数未变但中心指标仍出现差异，也证明运行间
采样噪声不可忽略。因此 size30 只能证明“值得继续验证”，不能作为上线结论，当前活动观看流
仍使用最近提交基线。

## 同序列逐 Track 重放

新增 `[MotionTrack]` 高精度诊断和 `tools/analysis/replay_size_filter.py`，在一次板端 RKNN/关联
结果上重放不同尺寸滤波参数。5570 个逐 Track 样本中包含 2979 个低速转换和 841 个运动转换；
重放的 max=0.45 基线与生产显示宽高的 P99 误差为 0.0005px、最大误差为 0.0008px，低于
0.10px 回归门限，说明重放器能够严格复刻生产尺寸路径。

固定 max=0.45/0.42/0.40/0.38/0.35/0.30 的扫描没有候选同时达到“低速边缘抖动改善至少 5%”
和“运动边缘误差增加不超过 10%”，因此不采用简单地全局降低 `track_size_alpha_max`。

## 中心运动门控候选

尺寸滤波改为由既有连续中心运动响应控制：低速时尺寸 alpha 上限可降到 0.30，运动增强时平滑
恢复到原来的 0.45，不增加硬阈值切换。公共配置默认关闭；专用观看配置才启用。

同一批 5570 个样本的 max=0.30 门控结果相对 max=0.45 基线：

- 低速边缘步长 RMS：0.403% → 0.360% 框对角线，改善约 10.7%；
- 全段边缘步长 P95：1.514% → 1.421%，改善约 6.1%；
- 运动边缘误差 P90：2.454% → 2.543%，增加约 3.6%；
- 运动边缘步长 P90：1.999% → 2.010%，基本不变。

板端 75 秒功能 Smoke 正常结束，4K/30FPS/18Mbps 稳定窗口约 28.39～31.05FPS，
`put_fail/out_drop/write_fail=0`；日志确认有效尺寸上限会随中心运动连续变化。随后已用关闭逐 Track
诊断日志的观看配置持续推送到固定地址 `rtmp://192.168.0.168/live/alertgateway`。该结果证明候选
生效且链路稳定，最终是否优于最近提交版本仍以慢速和快速移动画面的人工观察为准。

## 低速上限精细扫描与 size20 候选

在同一批 5570 个样本上继续扫描低速上限 0.30～0.12。0.18 是满足“运动边缘误差增加不超过
10%”的最后一个参数；0.16 开始超过门限。为避免贴着门限运行，选择 0.20 作为下一观看候选：

- 低速边缘步长 RMS：0.403% → 0.328% 框对角线，改善约 18.6%；
- 全段边缘步长 P95：1.514% → 1.328%，改善约 12.3%；
- 运动边缘误差 P90：2.454% → 2.621%，增加约 6.8%；
- 运动边缘步长 P90：1.999% → 2.026%，增加约 1.4%。

完整扫描见 `runs/testsrc2/motion_size_center_gated_full_sweep_20260716.*`。交叉构建、JSON、分析
单元测试和 diff 检查通过。size20 板端稳定窗口约 29.74～30.16FPS、约 17.8～18.2Mbps，
`put_fail/out_drop/write_fail=0`。另录制 20.01 秒固定输出，共 595 帧、约 17.87Mbps；全段解码
通过，未检测到超过 250ms 的冻结或黑帧。1FPS 全段和 10FPS 快速平移抽帧中，杯子、书本、
鼠标框连续跟随，未看到尺寸门控引入的突然缩放；目标框偶发出现/消失仍属于检测置信度变化。
该检查是自动与抽帧初验，不替代用户播放器中的最终视觉验收。

## 慢速中心平移优化

用户观看 size20 后确认慢速移动仍有抖动。逐 Track 数据显示，慢速样本中硬死区只占 3.2%，
死区释放步长 P90 约 2.75px，因此 6px 死区不是主要来源；慢速 `center_alpha` 的 P50/P90 仍为
0.243/0.366，单目标检测中心噪声仍会被较快吸收。

只压低单帧运动尖峰的重放虽能降低中心粗糙度，但快速运动误差明显增加，已拒绝。最终候选在
同一检测帧至少存在三个连续 Active Track 时，取所有原始中心位移的 x/y 中位数并以 0.25 平滑，
作为当前帧共同位移前馈；各框再用原有自适应滤波修正自身残差。该位移来自当前检测，不预测
未来位置；不足三个目标时完全回退原路径。

同一批 5570 个 Track 样本离线重放相对原路径：

- 低速中心粗糙度 RMS：0.766% → 0.690% 框对角线，改善约 9.9%；
- 低速中心误差 P90：2.354% → 2.235%，改善约 5.1%；
- 运动中心误差 P90：1.533% → 1.135%，改善约 26.0%；
- 全段中心步长 P95：5.772% → 5.666%，改善约 1.8%；
- 运动中心步长 P90：8.521% → 8.612%，增加约 1.1%。

65 秒板端 Smoke 正常 `Done`，4956 个逐 Track 样本中共同位移实际应用 3951 次（79.7%）；编码
FPS P50 为 29.98、码率 P50 为 18.03Mbps，失败计数为 0。随后关闭逐 Track 日志持续推流；另录制
20.01 秒、597 帧、约 17.97Mbps 输出，全段解码且未检出超过 250ms 的冻结或黑帧。抽帧未见共同
位移导致错框或突然跳转，但后续用户播放器观看确认框的晃动幅度更大、观感更不稳定，因此全局
共同位移候选（包括其一致性门控变体）已拒绝，代码与候选配置不保留。后续回到方向反转阻尼等
不引入每帧共同位移前馈的路线。
