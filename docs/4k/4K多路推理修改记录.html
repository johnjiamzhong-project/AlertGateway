五、问题与处理结果

遇到的问题

  • 四路4K并发画面卡顿
  • DMA-BUF导入RGA失败、花屏
  • 低处理帧率下PTS不均匀
  • 多路脚本复用输入流名

对应处理

  • 增加Pull分段性能统计
  • 优先NV12，NEON 2:1缩放
  • 使用源媒体PTS并做单调保护
  • 输入流独立命名并校验分辨率
1.2ms4K→1080p NEON缩放平均耗时，缩放不是主要瓶颈
27.6FPS四路全局NPU调度实测总处理能力，约每路6.9FPS
12–17FPS四路4K板端解码路径实测输出范围，仍受MPP/VPU资源限制

六、两种4K方案对比

当前板端降1080p

  • SRS转发4K原码流
  • 板端MPP解码4K
  • 解码后NV12缩放
  • 仍承担4K解码瓶颈

推荐的SRS主机转码

  • SRS接收4K原码流
  • 主机端FFmpeg转1080p
  • 板端只解码1080p
  • 降低板端VPU压力
4K源 → SRS(4K) → FFmpeg scale=1920:1080
    → SRS(1080p) → run_1080p专用板端管线

注意：接入SRS转码结果时，不能同时让本地测试脚本向同一1080p流名发布。
