## 2026-07-15 Mode 1 连续自适应滤波候选（待视觉验收）

- Mode 1 原先以中心距离 60 像素为界，在 `0.25/0.55` 两档 EMA 权重之间跳变；中心位置和
  框宽高还共用同一个权重。现改为连续自适应滤波，消除固定 4K 像素门限和权重突变。
- 运动量使用“相邻原始检测中心位移 / 目标框对角线 / 实际 PTS 间隔”计算，尺寸运动使用

## 下一阶段计划：仅用于关联的受限速度预测（尚未实现）

> 目标：减少画面移动时的 Track 重建和原始框直接重置，从源头降低边框跳变；预测结果只用于
> 检测关联，绝不进入叠框输出，因此不得引入预测残影或额外显示延迟。本节是实施计划，不代表
> 代码已经修改或验收通过。

### 一、保持不变的边界

- 保持 Mode 1 当前连续自适应显示滤波，不再降低中心或尺寸权重。
- 保持 `track_confirm_hits=2`、`track_ttl_ms=300`、`track_display_hold_ms=100` 和三帧视频
  PTS 对齐，不通过延长旧框显示时间掩盖关联失败。
- 保持 3840×2160、30FPS、18Mbps、4K 6 像素边框、MPP 编码和固定输出地址
  `rtmp://192.168.0.168/live/alertgateway`。
- 不启用 Mode 2 显示预测，不使用下一次检测插值，不修改 RKNN 推理、后处理、NMS 或模型阈值。

### 二、关联预测算法

1. 每条 Track 新增“关联专用速度”和其有效状态；显示框、关联框和原始检测框继续分开保存。
2. 只使用相邻两次已经确认关联的原始检测中心和真实 PTS 间隔估计速度。速度更新必须满足：
   类别一致、Track 为 Active、检测间隔有效、尺寸相似度满足现有关联约束，并且归一化速度未
   超过上限；不可信测量不能污染速度。
3. 速度采用 EMA 低通，初始建议权重 `0.40`。速度上限按目标框对角线归一化，初始建议每秒
   `2.5 × box_diagonal`，避免使用固定 4K 像素速度门限。
4. 下一批检测到来时，仅将 Track 的关联框中心平移 `velocity × dt`；预测时间最多 120ms，
   预测位移最多为当前框对角线的 25%。宽高不预测，仍取最后稳定框尺寸。
5. 用预测后的关联框计算类别、IoU、中心距离和尺寸相似度，再执行现有一对一贪心匹配。显示框
   不读取这个预测框，仍由当前检测与自适应滤波生成。
6. Track 进入 Shadow、连续失配、场景切换式大跳变或速度样本不可信时，速度立即清零或衰减；
   Tentative Track 在确认前不建立速度，防止单次误检产生错误预测。
7. 保留现有创新门限和重复框抑制。预测不能放宽类别约束，也不能让两个 Track 匹配同一检测。

- `association_dt_ms`：本次预测时间；
- `association_shift_px`：预测平移量；
- `association_prediction_limited`：是否触发时间或位移限幅；
- `matched_by_prediction`：原框不满足、预测框满足关联时为 1；
- `velocity_rejected_reason`：速度样本因间隔、尺寸、IoU 或速度上限被拒绝的原因。

必须在相同视频、相同起始位置和相同时长下对比当前基线。现有 10 分钟基线为：约 46,364 次
TrackUpdate、2,008 次新建轨迹、3,632 次 `center_alpha=1` 直接重置。首轮候选至少要求：

- 单位时间新建轨迹数和直接重置比例均下降 30% 以上；
- 没有同类别目标之间明显 ID 串换或框跳到其他目标；
- `pts_delta_ms` 不随时间增长，推理队列仍为 0/1；
- 稳定窗口输出不低于约 29.5FPS，`put_fail/out_drop/write_fail` 均为 0；
- 人工观察移动时抖动下降，同时没有预测越过目标、残影或新增跟随延迟。

### 六、实施与验收顺序

1. 先增加配置开关、Track 速度状态和诊断日志，保持开关默认关闭，完成编译和合成轨迹自测。
2. 开启关联预测，用固定视频运行短时板端 Smoke，确认无错误关联、崩溃和编码回归。
3. 使用同一输入片段做“关闭/开启”严格 A/B，分别统计 Track 新建、直接重置、预测匹配和限幅。
4. 指标达标后再启动固定地址供人工观察；主观观察未通过时不得只凭统计结果保留候选。
5. 若出现错误关联、框跳到其他目标、重建率未明显下降或任何新增显示延迟，关闭配置并回退到
   当前 Mode 1 自适应滤波，不继续扩大距离门限或 TTL。
